# CrediFi — Backend (Component B)

Node.js + TypeScript service for the CrediFi lending protocol. This slice is
the **database layer only** — Prisma schema, migration wiring, and a typed
client singleton. The scoring service, EIP-712 signer, on-chain event
listener, and REST API are built in subsequent slices on top of this.

Companion to [`@credifi/contracts`](../contracts) — deployed Oracle and Pool
addresses are at `contracts/deployments/hskTestnet.json`.

## Stack

- **Runtime**: Node.js 20+ (ESM)
- **Language**: TypeScript 5.5, strict mode
- **DB**: Neon (serverless Postgres) — see `DATABASE_URL` below
- **ORM**: Prisma 5

## Source of truth

**The chain is authoritative.** This database is a read cache and history
log used by the frontend's score-over-time chart, the recent-activity feed,
and the borrow/repay/liquidation history. It is never the source of truth
for funds or loan state — if the DB and the chain ever disagree, the chain
wins. The event listener's job is to keep this DB eventually consistent
with on-chain reality; the listener's `IndexerState.lastBlock` tracks how
far it has caught up.

## Prerequisites

- Node.js 20+
- A Neon Postgres database. Mayur owns the connection string — get it from
  the Neon dashboard under **Project → Connection Details** (use the
  **Pooled** connection string with `?sslmode=require`).

## Setup

```bash
cd backend
cp .env.example .env       # then paste your DATABASE_URL
npm install
npm run db:generate        # writes the typed @prisma/client to node_modules
npm run db:migrate         # generates the first migration AND applies it
                           # (requires DATABASE_URL to be set)
```

The first `npm run db:migrate` invocation will:

1. Diff `prisma/schema.prisma` against the current DB.
2. Generate `prisma/migrations/<timestamp>_init/migration.sql` with the
   `CREATE TABLE` statements for `wallets`, `scores`, `loans`, and
   `indexer_state` (and the indexes on them).
3. Apply that migration to your Neon DB.

**Commit the generated migration file** — that's how the schema version is
tracked. Future team members run `npm run db:migrate` and it just picks up.

## Scripts

| Script | What it does |
|---|---|
| `npm run db:generate` | Regenerate the `@prisma/client` types in `node_modules`. Run after any `schema.prisma` edit. |
| `npm run db:migrate` | `prisma migrate dev` — diff schema, create migration, apply to DB, regenerate client. **Dev only** — interactive. |
| `npm run db:deploy` | `prisma migrate deploy` — apply pending migrations without prompts. Use in CI / production. |
| `npm run db:studio` | Open Prisma Studio in a browser — browse / edit rows. |
| `npm run db:reset` | **Destructive.** Drop the DB and re-apply all migrations. |
| `npm run typecheck` | `tsc --noEmit` — strict typecheck against the generated client. |
| `npm run lint` | Placeholder. ESLint setup is in a follow-up slice. |

## Schema overview

| Model | Purpose |
|---|---|
| `Wallet` | One row per on-chain wallet we have observed. `address` is lowercased, indexed by PK. |
| `Score` | One row per computed score snapshot. `factors` is JSON so the breakdown can evolve without schema migrations. |
| `Loan` | One row per `Borrow` event, updated to `repaid`/`liquidated` on the corresponding event. |
| `IndexerState` | Singleton row. `lastBlock` is the highest block the event listener has processed. |

Full schema: `prisma/schema.prisma` (mirrors `PLAN.md` §5C exactly).

## Why money amounts are `String`, not `Float`

`principal` and `collateralLocked` store wei (uint256 on-chain). JavaScript
`Number` cannot safely represent integers beyond 2^53 (~9 × 10^15), and a
single wei value can be far larger. Using `Float` would silently lose
precision on most non-trivial amounts. Store wei as a decimal string and
convert at the boundary with `ethers.parseUnits` / `ethers.formatUnits`.

## Using the client

```ts
import { prisma } from "./db.js";

const wallet = await prisma.wallet.upsert({
  where: { address: addr.toLowerCase() },
  update: {},
  create: { address: addr.toLowerCase(), firstSeenAt: new Date() },
});

const activeLoans = await prisma.loan.findMany({
  where: { walletAddr: wallet.address, status: "active" },
});
```

The full API surface is generated from the schema and lives in
`node_modules/.prisma/client` — refer to Prisma's docs at
https://www.prisma.io/docs/orm/prisma-client for query patterns.
